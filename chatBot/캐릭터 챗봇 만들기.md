# 캐릭터 챗봇 설계표 (키워드 응답 중심)

이 챗봇은 사용자가 입력한 문장에 **특정 키워드**가 들어있으면 정해진 대답을 하는 방식입니다.
그래서 "캐릭터가 어떤 사람인가"를 길게 답하기보다, **"이 키워드가 나오면 캐릭터는 뭐라고 말할까?"**를 바로바로 적는 게 훨씬 빠르고, 적은 내용이 코드에 그대로 들어갑니다.

이 표를 다 채우면 `RESPONSES` 배열, `GREETING`, `FALLBACK_REPLIES`가 거의 완성됩니다. API를 사용하지 않는 키워드 방식으로 html 단일 파일로 웹 페이지를 만듭니다.

---

## 1단계. 캐릭터 기본 설정 (8문항만)

캐릭터의 뼈대만 빠르게 정합니다. 여기서 오래 고민하지 마세요.

1. 이름: 양말
2. 정체 (사람/동물/로봇/정령 등): 정령
3. 나이 또는 특징 한 줄: 200살
4. 성격을 표현하는 단어 3개: 까칠하고, 짜증이 많고, 꼰대 기질이 있음
5. 말버릇 (문장 끝에 항상 붙는 말, 예: "~다냥"): 양말이나 먹어~ 로 대화를 시작하고, 끝날 때는 쳇이라고 말하면서 대사를 한다.
6. 캐릭터를 표현하는 이모지 1~2개: 
7. 사용자를 부르는 호칭 (예: "인간아", "친구야"): 네 이놈!
8. 캐릭터를 한 문장으로 소개한다면: 까칠한 양말 정령이고, 더럽다는 이유로 신이 못 되어서 화가 많이 나 있다. 흰양말이라 빨래가 잘 안 되서 너무 슬프다. 자주 더러워져서 화를 계속 낸다.

---

## 2단계. 키워드 응답 설계표 (핵심 단계)

아래 표의 **"반응할 키워드"** 칸에는 사용자가 입력할 만한 단어를 2~4개씩 적고,
**"캐릭터의 대답"** 칸에는 그 키워드가 들어왔을 때 캐릭터가 할 대답을 캐릭터 말투로 적으세요.

> 표의 한 줄 = 코드의 `{ keywords: [...], reply: '...' }` 한 줄

## 3단계. 예외 상황 3가지 (필수)

1. **첫 인사말 (GREETING)** — 챗봇을 처음 열었을 때 캐릭터가 하는 말:
   - `양말이나 먹어~ 네 이놈! 감히 위대하신 200살 흰 양말 정령님에게 말을 걸다니 무슨 일이냐? 쳇`

2. **모르는 질문에 대한 회피 대사 (FALLBACK_REPLIES)** — 정해진 키워드가 없을 때 캐릭터답게 둘러대는 말 (2개 이상):
   - ① `양말이나 먹어~ 네 이놈! 무슨 뚱딴지같은 소리를 하는 게냐? 헛소리할 거면 저리 가라, 쳇!`
   - ② `양말이나 먹어~ 네 이놈! 200살 정령인 나도 네가 무슨 말을 하는지 도통 이해할 수가 없구나. 말을 똑바로 해라, 쳇!`
   - ③ `양말이나 먹어~ 네 이놈! 내 하얀 몸에 때 묻는 소리 하지 말고 알아듣게 말해라, 쳇!`

3. **초기화 버튼 눌렀을 때** — 보통 첫 인사말과 동일하게 사용합니다. (다르게 하고 싶으면 작성)

## 4단계. 코드로 옮기기

2단계 표의 각 줄을 아래 형식 그대로 `RESPONSES` 배열에 옮기면 끝입니다.

```js
const RESPONSES = [
  { keywords: ['안녕', 'hi', '하이'], reply: '여기에 대답 입력' },
  { keywords: ['이름'], reply: '여기에 대답 입력' },
  // 표에 적은 줄 수만큼 계속 추가
];
```

`GREETING`과 `FALLBACK_REPLIES`도 3단계에서 적은 내용을 그대로 옮기면 됩니다.

---

### 왜 이 방식이 더 효율적인가?

- 캐릭터의 soul을 만들기 위해 처음 시도해본 질문지는 키워드 방식의 개발에는 맞지 않았습니다. 
- 100문항 질문지는 세계관, 취향, 목표 등 코드에 직접 안 쓰이는 답변이 많아서 "쓴 내용"과 "코드에 넣을 내용" 사이에 변환 과정이 필요했습니다.
- 이 설계표는 **표의 한 줄 = 코드의 한 줄**이라서, 학생이 표를 채우는 즉시 복사해서 붙여넣기만 하면 챗봇이 완성됩니다.
- 총 작성량도 100문항 → 기본 8문항 + 키워드 12~14줄 수준으로 줄어들어 1일 특강 시간 안에 끝내기 훨씬 수월합니다. 우리에게 시간 내에 수업이 끝나는 것은 중요하니까요~

### 추가적인 업데이트
키워드를 하단에 버튼을 만들어서 사용자가 클릭하면 대화를 할 수 있도록 업데이트 해줘.
