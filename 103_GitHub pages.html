GitHub 세팅

실제 웹페이지로 접속하는 주소 (GitHub Pages)
HTML을 실제 웹사이트처럼 열려면 GitHub Pages를 활성화해야 해요.

활성화 방법:

  1. 저장소 → Settings → Pages
  2. Source를 main 브랜치로 설정 후 저장

그러면 주소가 이렇게 생성돼요:

https://유저명.github.io/저장소명/

https://luriluan0-a11y.github.io/gameContents

index.html은 기본 파일이라 파일명 없이 위 주소로 바로 접속돼요.

GitHub 활성화하기

Source를 main 브랜치로 설정 후 저장했는데 안 될 때는 저장소에 파일이 아직 없어서 그래요! 빈 저장소엔 GitHub Pages가 작동하지 않아요.

순서가 이래야 해요 👇

① 먼저 index.html 파일 업로드

화면 중간에 uploading an existing file 링크가 보이죠? 그걸 클릭해요!

그다음 만든 portfolio.html 파일을 index.html로 이름 바꾼 뒤 업로드 → Commit changes 클릭

② 파일 업로드 확인 후 GitHub Pages 설정

파일이 저장소에 올라간 걸 확인하고 나서

Settings → Pages → Branch: main → Save

③ 잠깐 기다리면 주소 생성!

1~2분 후 이 주소로 접속하면 돼요:

https://luriluan0-a11y.github.io/HTML_source/

💡 핵심은 순서예요! 파일 먼저 → Pages 설정 나중. 파일 없이 Pages 설정하면 배포가 안 돼요 😊

GitHub 폴더 만들기

GitHub는 빈 폴더를 만들 수 없어요. 파일을 넣어야 폴더가 생겨요!

방법: 파일 이름 앞에 폴더명을 써주면 돼요

  1. 저장소에서 Add file → Create new file 클릭
  2. 파일 이름 입력란에 이렇게 입력해요:
폴더명/파일명.txt

예시:

images/README.md
css/style.css
js/script.js
  1. / 를 입력하는 순간 자동으로 폴더가 만들어져요! ✨
  2. 파일 내용을 아무거나 입력하고 Commit changes 클릭

웹 업로드로 폴더째 올리는 방법

  1. Add file → Upload files 클릭
  2. 내 컴퓨터에서 폴더 자체를 드래그 앤 드롭
  3. 폴더 안의 파일들이 그대로 올라가요
💡 폴더 안에 파일이 하나도 없으면 GitHub에서 폴더가 사라져요. 항상 파일이 최소 1개 있어야 해요!
