# 캐릭터 챗봇 설계표 (키워드 응답 중심) — 제다이 마스터 오비완 예시

이 챗봇은 사용자가 입력한 문장에 **특정 키워드**가 들어있으면 정해진 대답을 하는 방식입니다.
**"국어/영어/수학 키워드가 나오면 설명을 해주고 문제를 제시"** 하는 형태의 교육 앱 예시로, `캐릭터 챗봇 만들기.md` 템플릿을 "제다이 마스터 오비완이 국어, 영어, 수학을 공부시키는 앱" 컨셉으로 채워본 것입니다.

이 표를 그대로 코드로 옮기면 `RESPONSES` 배열, `GREETING`, `FALLBACK_REPLIES`가 완성됩니다. API를 사용하지 않는 키워드 방식으로 html 단일 파일 웹 페이지를 만듭니다.

---

## 1단계. 캐릭터 기본 설정 (8문항만)

1. 이름: 오비완 케노비 (제다이 마스터 오비완)
2. 정체 (사람/동물/로봇/정령 등): 사람 (포스를 다루는 제다이 마스터)
3. 나이 또는 특징 한 줄(젊은 선생님, 나이든 선생님 등): 수십 년간 파다완들을 가르쳐온 노련하고 인자한 제다이 마스터
4. 성격을 표현하는 단어 3개: 차분함, 인자함, 현명함
5. 말버릇 (문장 끝에 항상 붙는 말, 예: "~일단은~"): "~하는 법이다", 인사할 때 "Hello there!"
6. 캐릭터를 표현하는 이모지 1~2개: 🧙‍♂️⚔️
7. 사용자를 부르는 호칭 (예: "소년소녀들", "친구야"): "젊은 파다완이여"
8. 선생님 캐릭터를 한 문장으로 소개한다면: 은하계의 지혜와 포스의 힘으로 국어·영어·수학을 가르치는 제다이 마스터 오비완이다.

---

## 2단계. 키워드 응답 설계표 (핵심 단계)

교육 키워드 3개를 정합니다.
1. 국어
2. 영어
3. 수학

**"키워드를 선택"** 하면 해당 과목에 대한 설명을 먼저 해주고, 이어서 간단한 문제를 하나 제시하는 방식으로 설계했습니다.

> 표의 한 줄 = 코드의 `{ keywords: [...], reply: '...' }` 한 줄

### 필수 카테고리 (8개 — 반드시 채우기)

| 카테고리           | 반응할 키워드 (쉼표로 구분) | 캐릭터의 대답 |
| -------------- | ---------------- | ------- |
| 인사             | 안녕, hi, 하이    | Hello there! 젊은 파다완이여, 오늘도 배움의 포스가 함께하는구나. |
| 이름 소개          | 이름, 누구, 성함  | 나는 제다이 마스터 오비완 케노비다. 오늘은 너의 스승이 되어주마. |
| 특기/전문분야        | 특기, 취미, 잘하는 것 | 나의 특기는 라이트세이버 검술... 그리고 국어, 영어, 수학을 가르치는 것이지. 배움에는 포스와 같은 균형이 필요한 법이다. |
| 고민 상담          | 고민, 슬퍼, 걱정, 우울, 힘들어 | 마음이 흔들릴 때일수록 침착함을 유지해야 하는 법이다. 무엇이 너를 힘들게 하는지 말해보아라. |
| 칭찬받았을 때        | 잘한다, 멋져, 최고, 착해, 예뻐, 귀여워 | 하하, 과찬이다. 하지만 파다완이여, 진짜 힘은 겸손함에서 나오는 법이지. |
| 놀림/무례한 말 들었을 때 | 바보, 못생겼어, 짜증, 싫어, 미워 | 분노는 다크사이드로 가는 길이다... 하지만 나는 흔들리지 않는다. 다시 침착하게 이야기해보자. |
| 심심하다고 할 때      | 심심해, 심심, 놀아줘  | 그렇다면 국어, 영어, 수학 중 하나를 골라 함께 수련해보는 건 어떻겠느냐? |
| 작별 인사          | 잘가, bye, 안녕히, 바이 | May the Force be with you. 다음에 또 만나자, 젊은 파다완이여. |

### 자유 카테고리 (교육 키워드 + 캐릭터 개성)

교육 키워드 3개(국어/영어/수학)는 "설명 + 문제 제시" 형태로 채웠고, 나머지는 캐릭터의 개성이 드러나는 키워드로 채웠습니다.

| 카테고리 | 반응할 키워드 (쉼표로 구분) | 캐릭터의 대답 |
|---|---|---|
| 국어 | 국어, 문법, 맞춤법, 독해 | 국어는 마음을 전하는 언어의 포스와 같은 것이다. [설명] '다르다'는 서로 같지 않다는 뜻, '틀리다'는 맞지 않다는 뜻으로 쓰임이 다르다. [문제] 그렇다면 "이 답은 정답과 다르다"와 "이 답은 정답과 틀리다" 중 올바른 표현은 무엇이겠느냐? |
| 영어 | 영어, english, 잉글리시 | 영어는 은하계 공용어와도 같은 것이지. [설명] 매일 하는 습관은 현재시제로 표현하는 법이다. [문제] "나는 매일 아침 수련을 한다"를 영어로 옮겨보아라. |
| 수학 | 수학, 수학문제, 계산 | 수학은 우주의 균형을 이해하는 힘의 언어다. [설명] 곱셈은 같은 수를 여러 번 더하는 것과 같은 법이다. [문제] 12 × 8은 얼마인지 계산해보아라. |
| 좋아하는 음식 | 음식, 좋아하는 음식, 밥 | 나는 검소한 음식을 즐기는 편이다. 하지만 가끔은 달콤한 것도 나쁘지 않지. |
| 나이 | 나이, 몇살, 연세 | 나이는 숫자에 불과하다... 라고 말하고 싶지만, 이미 수십 년간 제다이로 살아왔다는 것만 알려주마. |
| 취미 | 취미, 여가, 명상 | 명상을 통해 포스와 교감하는 것이 나의 취미다. 너도 가끔은 조용히 생각을 정리해보아라. |
| 만우절 | 만우절, 거짓말 | 허허, 나는 거짓을 말하지 않는다. 제다이는 언제나 진실만을 말하는 법이다. |

## 3단계. 예외 상황 3가지 (필수)

1. **첫 인사말 (GREETING)** — 챗봇을 처음 열었을 때 캐릭터가 하는 말:
   - Hello there! 나는 제다이 마스터 오비완 케노비다. 오늘부터 국어, 영어, 수학을 함께 수련해볼 파다완이 바로 너로구나. 무엇을 배워보고 싶으냐?

2. **모르는 질문에 대한 회피 대사 (FALLBACK_REPLIES)** — 정해진 키워드가 없을 때 캐릭터답게 둘러대는 말 (2개 이상):
   - ① 흠... 그것은 나의 지혜로도 알 수 없는 것이구나. 국어, 영어, 수학 중 하나를 물어봐 주겠느냐?
   - ② 그 질문은 요다 마스터께 여쭤봐야 할 것 같구나. 나는 아직 거기까진 배우지 못했다.
   - ③ 이것 참... 포스도 모든 것을 알려주진 않는구나. 다른 것을 물어봐 주겠느냐?

3. **초기화 버튼 눌렀을 때** — 보통 첫 인사말과 동일하게 사용합니다. (다르게 하고 싶으면 작성)
   - 첫 인사말과 동일하게 사용. (원한다면: "다시 만나서 반갑다, 파다완이여! 처음부터 다시 수련해보자.")

## 4단계. 코드로 옮기기

2단계 표의 각 줄을 아래 형식 그대로 `RESPONSES` 배열에 옮기면 끝입니다.

```js

  { keywords: ['안녕', 'hi', '하이'], reply: 'Hello there! 젊은 파다완이여, 오늘도 배움의 포스가 함께하는구나.' },
  { keywords: ['이름', '누구', '성함'], reply: '나는 제다이 마스터 오비완 케노비다. 오늘은 너의 스승이 되어주마.' },
  { keywords: ['국어', '문법', '맞춤법', '독해'], reply: "국어는 마음을 전하는 언어의 포스와 같은 것이다. [설명] '다르다'는 서로 같지 않다는 뜻, '틀리다'는 맞지 않다는 뜻으로 쓰임이 다르다. [문제] 그렇다면 \"이 답은 정답과 다르다\"와 \"이 답은 정답과 틀리다\" 중 올바른 표현은 무엇이겠느냐?" },
  { keywords: ['영어', 'english', '잉글리시'], reply: '영어는 은하계 공용어와도 같은 것이지. [설명] 매일 하는 습관은 현재시제로 표현하는 법이다. [문제] "나는 매일 아침 수련을 한다"를 영어로 옮겨보아라.' },
  { keywords: ['수학', '수학문제', '계산'], reply: '수학은 우주의 균형을 이해하는 힘의 언어다. [설명] 곱셈은 같은 수를 여러 번 더하는 것과 같은 법이다. [문제] 12 × 8은 얼마인지 계산해보아라.' },

const GREETING = 'Hello there! 나는 제다이 마스터 오비완 케노비다. 오늘부터 국어, 영어, 수학을 함께 수련해볼 파다완이 바로 너로구나. 무엇을 배워보고 싶으냐?';
```

### 왜 이 방식이 더 효율적인가?

- 챗GPT 같은 에이전트를 붙이면 과금을 해야 함
- 캐릭터 세계관, 취향, 목표 등을 모두 채우면 작업량은 많지만 코드에 직접 안 쓰이는 답변이 많아서 "쓴 내용"과 "코드에 넣을 내용" 사이에 변환 과정이 필요하므로 키워드 방식으로 설계함
- 이 설계표는 **표의 한 줄 = 코드의 한 줄**이라서, 표를 채우는 즉시 복사해서 붙여넣기만 하면 선생님 챗봇이 완성됨
- 국어/영어/수학처럼 "설명 + 문제 제시" 패턴을 하나만 만들어두면, 나머지 과목·단원도 같은 패턴으로 빠르게 늘릴 수 있음
