CSS align-items: flex-end와 end의 차이점

CSS Flexible Box Layout(Flexbox)과 Grid Layout에서 align-items: flex-end;와 align-items: end;는 외형상 유사하게 작동하는 것처럼 보이지만, 지원하는 레이아웃 시스템과 기준이 되는 축(Axis)에서 중요한 차이점이 있습니다.

1. 한눈에 보는 핵심 차이

특징 align-items: flex-end; align-items: end;
주요 대상 Flexbox (display: flex;) Grid / Block / Alignment 규격
정렬 기준 Flex 라인의 flex-end (교차 축의 끝) 글자 흐름/방향에 따른 block-end (끝)
Grid에서의 동작 Flexbox 전용이므로 Grid에서는 작동하지 않음 Grid 셀 안에서 완벽하게 작동함
호환성 구형 브라우저까지 폭넓게 지원 비교적 최신 스펙 (현대 브라우저는 모두 지원)

2. 왜 두 개가 따로 존재할까?

💡 flex-end (Flexbox 전용)

  • Flexbox 레거시: Flexbox가 처음 등장했을 때, 정렬 방향은 부모의 flex-direction에 따라 유연하게 바뀌어야 했습니다. 그래서 Flexbox만의 고유한 기준점인 flex-start와 flex-end가 만들어졌습니다.
  • 작동 원리: flex-direction: column;이나 row;에 따라 교차 축(Cross Axis)의 시작점과 끝점을 기준으로 아이템을 정렬합니다.

💡 end (CSS Box Alignment 표준)

  • 새로운 표준 규격: 이후 CSS에 Grid 레이아웃이 도입되면서, Flexbox뿐만 아니라 전반적인 CSS 레이아웃에 공통으로 쓸 수 있는 통합 정렬 규칙(CSS Box Alignment Module)이 필요해졌습니다. 이때 도입된 범용 키워드가 바로 end입니다.
  • 작동 원리: 글자 쓰기 방향(Writing Mode)의 블록 축(Block Axis) 끝점을 기준으로 정렬합니다. 보통 위에서 아래로 글을 쓰는 한국어/영어 환경에서는 '아래쪽'이 됩니다.

3. 실무에서의 실질적인 차이점 (주의할 점)

  • Flexbox 컨테이너 안에서: 현대 브라우저(Chrome, Safari, Firefox 등)는 display: flex; 환경에서 align-items: end;를 써도 flex-end처럼 알아서 해석해 줍니다. 하지만 아주 오래된 구형 브라우저나 일부 환경에서는 end를 인식하지 못할 수 있으므로, Flexbox를 쓸 때는 flex-end를 쓰는 것이 가장 안전합니다.
  • Grid 컨테이너 안에서: display: grid; 환경에서는 flex-end를 사용할 수 없습니다. Grid에서는 반드시 end를 사용해야 감싸고 있는 셀(Cell)의 아래쪽으로 아이템이 정렬됩니다.

📌 요약하자면!

  • Flexbox(display: flex;)를 쓸 때는 안전하게 flex-end를 쓰세요.
  • Grid(display: grid;)를 쓸 때는 표준 스펙인 end를 쓰세요.
